# Nikke Union Raid Planner

A toolkit for planning and reporting optimal hit assignments in **Nikke: Goddess of Victory — Union Raid Hard Mode**.

---

## Requirements

- Python 3.9+
- `pulp` (optional — enables the exact ILP solver)

```bash
pip install pulp
```

Without `pulp`, all tools fall back to the built-in greedy solver automatically.

---

## Data Files

All data files live in `data/`.

### `data/bosses.csv` (or `bosses_template.csv`)

Boss HP values and element weaknesses.

| Column | Description |
|--------|-------------|
| `boss_name` | Display name of the boss |
| `hp_level1` | HP for Level 1 |
| `hp_level2` | HP for Level 2 |
| `hp_level3` | HP for Level 3 |
| `element_weakness` | Element the boss is weak to (`iron`, `water`, `electric`, `fire`, `wind`) |

### `data/teams.csv`

Member team submissions used by the optimizer.

| Column | Description |
|--------|-------------|
| `member` | Member name |
| `boss_name` | Boss this team targets |
| `damage` | Damage dealt in the mock run |
| `unit1`–`unit5` | The five units in the composition |

> Do not edit this file manually — generate it from `teams.txt` using `convert_teams.py`.

## Tools

### 1. `tools/convert_teams.py` — Convert teams export to CSV

Converts the guild's raw tab/space-separated teams export (`teams.txt`) into the `teams.csv` format expected by the other tools.

**Element → boss name mapping is read automatically from the bosses CSV.**

```bash
# Default paths
python tools/convert_teams.py

# Custom paths
python tools/convert_teams.py --txt teams.txt --bosses data/bosses_template.csv --out data/teams.csv
```

| Argument | Default | Description |
|----------|---------|-------------|
| `--txt` | `data/teams.txt` | Input `.txt` export file |
| `--bosses` | `data/bosses_template.csv` | Bosses CSV (for element-to-boss mapping) |
| `--out` | `data/teams.csv` | Output CSV file |

**Input column layout** (tab or 2+ space separated):

```
member  rank  element  run#  damage  unit1  unit2  unit3  unit4  unit5  [flag]  [secondary]
```

Columns `rank`, `run#`, `flag`, and `secondary` are ignored during conversion.

---

### 2. `tools/nikke_optimizer.py` — Hit assignment optimizer

Assigns member team compositions to boss levels, maximising guild progression while respecting:

- Per-member unit uniqueness (a unit cannot appear twice in the same member's runs)
- Per-member hit limit (default: 3 hits per member)
- Boss level gating (all 5 bosses must clear L1 before L2 unlocks, etc.)
- Damage capping (excess damage beyond boss HP is noted as wasted)

```bash
# Basic usage
python tools/nikke_optimizer.py --bosses data/bosses.csv --teams data/teams.csv

# Apply a 5% damage buffer (for when members perform better in the real fight)
python tools/nikke_optimizer.py --buffer 5

# Force a specific solver
python tools/nikke_optimizer.py --solver greedy
python tools/nikke_optimizer.py --solver ilp

# Run the built-in self-test
python tools/nikke_optimizer.py --test
```

| Argument | Default | Description |
|----------|---------|-------------|
| `--bosses` | `data/bosses.csv` | Path to boss HP CSV |
| `--teams` | `data/teams.csv` | Path to member teams CSV |
| `--solver` | `auto` | `auto`, `ilp` (requires pulp), or `greedy` |
| `--max-hits` | `3` | Maximum hits allowed per member |
| `--weights` | `1000 100 10` | Score weights for L1 / L2 / L3 progress |
| `--time-limit` | `60` | ILP solver time limit in seconds |
| `--buffer` | `0` | Scale all damages up by this percentage before optimizing |
| `--test` | — | Run built-in self-test and exit |

**Solvers:**

| Solver | Requires | Quality |
|--------|----------|---------|
| `greedy` | Nothing (stdlib only) | Near-optimal |
| `ilp` | `pip install pulp` | Proven optimal per stage |
| `auto` | — | Uses ILP if available, greedy otherwise |

Output is printed to stdout as a formatted text plan.

### 3. `tools/nikke_html_report.py` — HTML report generator

Runs the optimizer and generates a self-contained HTML report with visual progress bars, assignment tables, member statistics, and wasted damage summaries.

```bash
# Default paths — writes report.html
python tools/nikke_html_report.py --bosses data/bosses.csv --teams data/teams.csv

# Custom output file
python tools/nikke_html_report.py --output my_report.html

# With damage buffer
python tools/nikke_html_report.py --buffer 5
```

| Argument | Default | Description |
|----------|---------|-------------|
| `--bosses` | `data/bosses.csv` | Path to boss HP CSV |
| `--teams` | `data/teams.csv` | Path to member teams CSV |
| `--solver` | `auto` | Same as optimizer |
| `--max-hits` | `3` | Maximum hits per member |
| `--weights` | `1000 100 10` | Score weights for L1 / L2 / L3 |
| `--time-limit` | `60` | ILP solver time limit in seconds |
| `--buffer` | `0` | Damage buffer percentage |
| `--output` | `report.html` | Output HTML file path |

**Report sections:**

| Section | Contents |
|---------|----------|
| Meta strip | Generation date, solver used, score, runs assigned, member count, raid status |
| Boss Progress Overview | Per-boss cards with L1/L2/L3 progress bars and wasted damage |
| Assignments by Boss & Level | Per-level tables showing member, damage, units, and a wasted damage footer |
| Member Statistics | Hit usage indicator, boss targets, and total damage per member |
| Notices | Members with unused hits, dropped teams, uncovered bosses |

The output is a **single portable `.html` file** with no external dependencies.

---

## Typical Workflow

```bash
# 1. Convert the guild's teams export
python tools/convert_teams.py --txt teams.txt

# 2. Generate the HTML report
python tools/nikke_html_report.py --bosses data/bosses_template.csv --teams data/teams.csv

# 3. Open report.html in a browser
```
